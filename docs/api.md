# API bridge

Все методы, кроме `/health`, требуют заголовок `Authorization: Bearer YOUR_API_TOKEN`.

## `GET /health`

Проверяет работу HTTP-процесса: `{"ok": true}`.

## `GET /status`

Возвращает подключение, состояние, заряд, мощность, ошибку и версию прошивки.

## `POST /command`

```json
{"command": "clean"}
```

| Команда | Действие |
|---|---|
| `clean` | Начать автоматическую уборку |
| `stop` | Остановить |
| `dock` | Вернуться на базу |
| `fan_normal` | Обычная мощность |
| `fan_strong` | Высокая мощность |
| `locate` | Отправить команду поиска |

Поле `ok=true` означает, что bridge увидел ожидаемое состояние. Для `dock` добавлена отложенная повторная проверка, потому что спящий робот может кратковременно подтвердить команду без движения.

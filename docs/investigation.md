# Технический отчёт о восстановлении управления ABIR X6

## Симптом

Робот ABIR X6 ранее работал через WeBack, затем перестал управляться: первичная настройка Wi-Fi завершалась, робот появлялся в приложении, но отображался `Offline`; локальное управление кнопками и штатным пультом продолжало работать.

## Что проверялось

### Домашняя сеть

Использовалась отдельная сеть 2.4 ГГц. Проверялись пароль, DHCP, DNS, интернет и VPN-маршрутизация. Робот появлялся среди клиентов роутера. Вывод: Wi-Fi-модуль и домашняя сеть работоспособны.

### WeBack

Проверялись повторная привязка, разные регионы и новые аккаунты, включая Китай и Сингапур. Робот оставался `Offline`. Смена региона не исправляет проблему для проверенного экземпляра.

### Сетевой трафик и прошивка

Исследование показало, что робот использует AWS IoT; в прошивке присутствуют состояния `AutoClean`, `Standby`, `BackCharging` и другие; основные команды можно передавать через device shadow; карта и история не возвращаются тем же простым каналом.

## Рабочий путь

После совместимой привязки робот стал доступен в Redmond/grit-cloud. Были определены API авторизации, региональный endpoint, уникальный `thing_name`, `sub_type`, формат чтения состояния и отправки команд.

На VPS был запущен bridge, который авторизуется в облаке, обновляет сессию, читает состояние, отправляет команды в AWS IoT shadow, проверяет их выполнение и предоставляет собственный API с Bearer-токеном.

## Найденные состояния

| Значение | Наблюдаемое значение |
|---|---|
| `AutoClean` | Автоматическая уборка |
| `Standby` | Остановлен/ожидает |
| `BackCharging` | Возвращается на базу |
| `Charging` / `PileCharging` | Заряжается |
| `ChargeDone` | Зарядка завершена |
| `Hibernating` | Сон; иногда глубокий сон |

## Почему нужна повторная проверка

Спящий робот мог получить `BackCharging`, сообщить `BackCharging`, не включить двигатели и через несколько секунд снова сообщить `Hibernating`. Поэтому bridge повторно проверяет состояние и не считает краткое изменение статуса успешным выполнением.

## Глубокий сон

Проверенный X6 просыпается от физической кнопки и штатного ИК-пульта, но не всегда от облачной команды. Вероятно, Wi-Fi-модуль остаётся активным, а основной контроллер и двигатели спят; ИК-приёмник и кнопка имеют аппаратный путь пробуждения.

## Что не решено

- Надёжное сетевое пробуждение из глубокого сна.
- Живая карта и история маршрутов.
- Звуковая команда поиска на проверенном экземпляре.

Возможный обход глубокого сна: обучаемый ИК-передатчик рядом с роботом, затем основная команда через bridge.
